# Artifact Lineage

Debrief is concerned with artifact lineage, not just final outputs.

## Why Lineage Matters

An artifact is more useful when the system can explain:

- where it came from
- what workflow produced it
- what inputs it depended on
- what transitions occurred before publication
- whether it is authoritative for downstream use

Lineage is part of what makes work resumable and reviewable rather than merely stored.

## Public Boundary

This repository does not include private runtime state files, production schemas, or raw internal artifact histories. The public objective is conceptual clarity, not disclosure of private operational records.

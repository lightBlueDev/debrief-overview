# Workspace Relationships

Debrief is developed within a broader private workspace concerned with governed, continuous, AI-assisted implementation.

## Relationship To Adjacent Systems

The surrounding context helps explain Debrief's boundaries:

- Debrief focuses on continuity runtime and orchestration truth
- Proof-of-Care focuses on governed action boundaries
- PACE focuses on planning, audit, handoff, and closure discipline

This separation is architectural, not cosmetic.

Debrief is not the governance layer and not the methodology layer. It is the runtime layer.

This public repository presents Debrief as the public-facing identity for that continuity-runtime work.

## Why That Matters Publicly

Acknowledging the larger workspace makes the project more legible without requiring the public repository to absorb the full private ecosystem.
